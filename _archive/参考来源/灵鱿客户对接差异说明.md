## 1. 文档定位

本文档说明：**我方内部标准**与**灵鱿客户协议**之间的差异，以及对外承诺边界。

| 文档 | 角色 |
| ---- | ---- |
| 灵鱿 PDF / 重点版 | 客户侧联调最小集 |
| 内部标准版 | 我方云 + 车端实际实现依据 |
| 本文档 | 两者之间的「翻译层」，供商务沟通与验收对齐 |

**对外统一口径**：

> 我方云平台完整支持《车辆 IOT 实现方案》MQTT 标准接口，并兼容灵鱿联调所需最小指令集；导航、建图、高精度定位由车端软件完成，云平台不参与路径规划与地图管理。

## 3. 关键字段差异与映射

### 3.1 position（位置）

| 项目 | 灵鱿 PDF / 重点版 | 我方内部标准 |
| ---- | ----------------- | ------------ |
| 格式 | `"x,y,yaw"`（米、弧度） | `"经度,纬度"` |
| 示例 | `"114.05,22.50,3.14"` | `"114.05129,22.508236"` |

**对接方案**：

- 云平台存储与展示：**经度,纬度**（WGS84 或项目约定坐标系）。
- 车端 MQTT 适配层负责与灵鱿格式的**双向转换**（若客户平台仍发送 x,y,yaw）。
- 商务说明：*「位置信息语义一致，传输格式由车端适配层统一转换，客户无需改造已有解析逻辑。」*

### 3.2 workStatus（作业状态）

| 值 | 灵鱿 PDF / 重点版 | 我方内部标准 | 映射建议 |
| -- | ----------------- | ------------ | -------- |
| 0 | 空闲 | 空闲 | 一致 |
| 1 | 任务中 | 任务中 | 一致 |
| 2 | **暂停中** | **故障** | 需映射：我方 fault→客户侧展示为故障，非暂停 |
| 3 | **故障中** | **充电** | 需映射 |
| 4 | **开机初始化中** | **急停** | 需映射 |

**对接方案**：

- 联调前提供 **workStatus 映射表**（Excel/附录），双方测试用例对齐。
- 若客户 UI 依赖「暂停=2」，可通过车端上报前转换，或客户侧按映射表解析。

### 3.3 2010001 任务下发

| 项目 | 灵鱿 PDF / 重点版 | 我方内部标准 |
| ---- | ----------------- | ------------ |
| taskNodes | 含 x/y/z/w 坐标 | **仅站点名 + duration** |
| returnPoint | tasktype + 坐标结构 | **返航点名称字符串** |
| taskSteps | 未定义 | 云不支持，车端可选 |

**对接方案**：

- 云平台任务编排界面：**只维护站点名称**，不维护坐标。
- 车端维护「站点名 ↔ 坐标」点位库；收到 2010001 后本地解析并执行导航。
- 若客户平台**必须下发坐标**：由车端 MQTT 适配层接收并转换，**不要求云平台存储或编辑坐标**。
- 商务说明：*「任务下发接口兼容灵鱿 格式；坐标数据可在车端落地，云平台以站点名称驱动任务编排。」*

### 3.4 故障 type 编号

| 文档 | 故障 type |
| ---- | --------- |
| 灵鱿 重点版（笔误） | 1010003 |
| 灵鱿 PDF / 我方标准 | **1010004** |

**对接方案**：联调与验收统一使用 **1010004**；书面纠正重点版笔误。

## 4. 联调最小集 vs 完整能力

### 4.1 灵鱿联调最小集（客户 PDF 范围）

客户 PDF 定义的联调 type：

```
上报：1010001、1010002、1010004
下发：2010001、2010002、2010004、2010005、2010008
```

### 4.2 我方建议联调范围（灵鱿 项目）

在最小集基础上，建议纳入：

| type | 理由 |
| ---- | ---- |
| 2010009 | 运营暂停/恢复，客户运维常用 |
| 2010007 | 点位名称同步，与任务编排配合 |

不纳入云平台联调：

| type | 理由 |
| ---- | ---- |
| 1010003/5 | 导航相关 |
| 2010010~12 | 建图/定位/地图，车端本地 |

## 5. 验收建议

### 5.1 必测项（P0）

- [ ] 车辆注册、MQTT 连接、上下线
- [ ] 1010001 状态 1Hz 上报（position 格式按映射表）
- [ ] 2010001 任务下发（站点名模式）及 dev/reply
- [ ] 2010004 视频开关 + 周期心跳
- [ ] 2010005 遥控（无需 reply）
- [ ] 2010008 紧急停止 / 解除
- [ ] 1010002 任务列表上报 + 云确认

### 5.2 建议测项（P1）

- [ ] 1010004 故障上报与告警
- [ ] 2010009 任务暂停 / 恢复 / 停止
- [ ] 2010002 target 控制（GIMBAL / AUDIO，见内部标准 §9.2）

### 5.3 不验收项（车端能力）

- 建图、地图同步、定位校准
- 10Hz 高频定位
- 云平台维护 taskNodes 坐标
- taskSteps 组合任务编排 UI

## 6. 商务话术参考

### 6.1 客户问：「为什么云平台不下发坐标？」

> 任务坐标属于车端导航数据，由车辆本地点位库统一管理，可保证离线/弱网场景下任务仍可执行。云平台通过站点名称下发任务，与 IOT 行业标准一致，同时车端适配层可兼容灵鱿 协议中的坐标字段。

### 6.2 客户问：「workStatus 枚举不一致怎么办？」

> 双方在联调前确认映射表；车端可在上报前做枚举转换，确保客户监控大屏正常展示，无需改动云平台核心逻辑。

### 6.3 客户问：「是否支持 AGV 文档里的建图/地图同步？」

> 建图与地图管理属于车端自动驾驶软件能力，不在云平台 MQTT 接口范围内。如需远程运维，可单独评估运维工具或 VPN 方案，不纳入标准 MQTT 协议验收。

### 6.4 客户问：「1010003 是故障还是定位？」

> 以 1010004 作为故障上报 type；1010003 不在标准联调范围内。重点版 docx 中 1010003 标注为故障属于文档笔误，联调以 PDF 及本文档为准。
