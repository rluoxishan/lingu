## 一、出发前今晚必做（约 20 分钟）

### 1.1 准备配置（MQTT 模式无需云账号）

现场 zip 内 `config/site/server.yaml` 已设 **`dataSource: mqtt`**。  
需在中转机安装 **EMQX**（或 Mosquitto），并确认 `config/site/mqtt.yaml` 中 `brokerUrl` 正确。

`.env` **仅 cloud 模式需要** `CLOUD_*`；MQTT 模式可不带 `.env`。

### 1.2 打 U 盘离线包（开发机有网）

```powershell
cd d:\LINGUSET\beidou-bridge
PREPARE-offline-usb.bat
```

或：

```powershell
npm run prepare:offline-usb
```

另在开发机下载并拷 U 盘：**Node.js 20 .msi**、**Mosquitto Windows 安装包**。

### 1.3 物理携带清单（U 盘）

| # | 物品 | 说明 |
|---|------|------|
| 1 | **部署 zip** | `release/beidou-bridge-site-*.zip`（含 node_modules，中转机无需 npm） |
| 2 | **Node.js 20 .msi** | 中转机若无 Node |
| 3 | **Mosquitto .exe** | 中转机 MQTT Broker |
| 4 | **中转机离线部署手册** | `docs/中转机离线部署手册-MQTT模式.md`（建议打印） |
| 5 | zip 副本（可选） | 5G 模拟电脑跑 `SIMULATE-vehicle.bat` |
| 6 | 网线/专网信息 | 中转机 IP、5G 能否 ping 通 |

**不需要**：`.env`、云账号、外网。

### 1.4 出发前向北斗方确认（微信/邮件发即可）

请北斗方联调前提供：

| # | 项 |
|---|-----|
| 1 | 中转机 **内网 IP**（Windows，能装 Node.js 20） |
| 2 | 中转机 → `https://sztu.lingubot.cn` **能上网** |
| 3 | 北斗 → 中转机 **8080 端口互通** |
| 4 | 北斗 **回调 URL**（register 里填的 url，例 `http://x.x.x.x:端口/callback`） |
| 5 | 联调 **时间段** + 现场联系人 |

**发给北斗的接口地址模板**（`<IP>` 换成中转机 IP，到场填）：

```
健康检查:  GET  http://<IP>:8080/health
注册回调:  POST http://<IP>:8080/api/v1/beidou/callback/register
反控(可选): POST http://<IP>:8080/api/v1/beidou/navigation
```

register 请求 Body 示例：

```json
{
  "url": "http://<北斗回调地址>",
  "frequency": 4000
}
```

## 二、到现场后最少操作（约 15 分钟接真北斗）

> **中转机一般由北斗方提供。** 以下在 **中转机** 上操作。

### 步骤 0：填写环境表（1 分钟）

| 项 | 填写 |
|----|------|
| 中转机 IP | |
| health | `http://____:8080/health` |
| register | `http://____:8080/api/v1/beidou/callback/register` |
| 北斗回调 URL | |

---

### 步骤 1：解压 + 放 `.env`（2 分钟）

1. 解压 zip 到例如 `D:\beidou-bridge`
2. 把带来的 **`.env`** 复制到 `D:\beidou-bridge\`（与 `dist` 同级）
3. 若机器 **没有 Node.js**：安装 [Node.js 20 LTS](https://nodejs.org/)，装完 **重开** PowerShell

目录应类似：

```
D:\beidou-bridge\
  .env                 ← 必须有
  dist\main.js         ← 必须有
  config\site\         ← 现场配置
  node_modules\        ← zip 内已含
  现场-启动中转.bat
  现场-检查环境.bat
```

---

### 步骤 2：检查环境（2 分钟）

双击 **`CHECK-env.bat`**（或 `现场-检查环境.bat`）

- 全部 `[OK]` → 继续  
- 有 `[FAIL]` → 看下方「常见问题」，修完再查

---

### 步骤 3：开放防火墙（1 分钟，北斗从别的机器访问时必做）

**右键以管理员运行** → **`OPEN-firewall-8080.bat`**

---

### 步骤 4：启动中转（1 分钟）

双击 **`START-bridge.bat`**（或 `现场-启动中转.bat`）

窗口保持打开，应看到类似：

```
[site] listening on 0.0.0.0:8080
```

另开 PowerShell 验证：

```powershell
Invoke-RestMethod http://127.0.0.1:8080/health
```

应返回：`status = up`

---

### 步骤 5：让孟泽启车（并行）

联系 **孟泽**：联调当天 **LU2606000100 商用版** 启动。

你可双击 **`QUERY-vehicle.bat`** 确认云上有：

- `workStatus`、`battery` 有值  
- `position_xyz` 为 0 **不挡 B1**，有坐标更好

---

### 步骤 6：把 register 地址给北斗（1 分钟）

告诉北斗方：

```
POST http://<中转机IP>:8080/api/v1/beidou/callback/register
Content-Type: application/json

{
  "url": "<北斗方提供的回调URL>",
  "frequency": 4000
}
```

**由北斗方发起 register**（或你在本机自测，把 url 换成北斗真实回调）：

```powershell
$body = '{"url":"http://<北斗回调URL>","frequency":4000}'
Invoke-RestMethod -Method POST `
  -Uri "http://127.0.0.1:8080/api/v1/beidou/callback/register" `
  -ContentType "application/json" -Body $body | ConvertTo-Json -Depth 6
```

**预期 register 响应：**

- HTTP 200  
- `code = 0`  
- `data.vehicleIds` 含 **`LU2606000100`**

---

### 步骤 7：B1 验收（等 4～8 秒）

| # | 检查 | 谁看 | 预期 |
|---|------|------|------|
| R1 | register | 你 | code=0，vehicleIds 含 LU2606000100 |
| R2 | 北斗收到 POST | 北斗 | 约每 4s 一次，Body 含 data.vehicleId |
| R3 | 北斗 HTTP 响应 | 北斗 | **code: 1000** |
| R4 | 中转日志 | 你 | `[scheduler] pushed vehicle=LU2606000100` |

**B1 通过 = R1～R4 全 OK。** 坐标为 0 不挡验收。

---

### 步骤 8（可选）：重启恢复测试

1. 中转窗口 **Ctrl+C** 停止  
2. 再双击 **`START-bridge.bat`**（或 `现场-启动中转.bat`）  
3. **不重新 register**，北斗仍应收到推送（配置在 `data-site/beidou-callback.json`）

## 四、常见问题（现场自助）

| 现象 | 处理 |
|------|------|
| 没有 Node.js | 安装 Node 20 LTS，重开窗口 |
| `.env` 登录 401 | 检查账号密码；租户名是否为「超级管理员」 |
| register 502 | `.env` 错，或 `config/site/vehicles.yaml` 无 LU2606000100 |
| 8080 被占用 | 任务管理器结束旧 node 进程，或改 `config/site/server.yaml` port |
| 北斗访问不了 health | 防火墙未开 / IP 错 / 不在同一网段 |
| 北斗收不到推送 | register 的 url 错；中转未启动；看 R1 是否成功 |
| scheduler non-success | 北斗 POST 响应须 **code:1000** |
| position_xyz 全 0 | 孟泽启商用版；**不挡 B1** |
| navigation 超时 | B2 已知问题，本次不测 |

## 七、没有 AI 时怎么办

1. **只按本文 §二 的 8 步操作**，双击根目录 bat 即可  
2. 卡住时对照 **§四 常见问题**  
3. 需要 Mock 预演（北斗未到前）：`npm run sim:lab`（需先 `npm ci` 且有效 `.env`）  
4. 仍无法解决：拍 **中转窗口日志** + **register 响应 JSON** + **北斗侧现象**，发我方开发

**不要用腾讯文档/WPS 改程序目录里的文件；只改 `.env` 和 `config/site/vehicles.yaml`。**
