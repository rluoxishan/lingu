# 0629 协议 — 中转侧两段响应定稿

> **用途**：同步《中转系统与北斗系统交互协议0630.docx》中接口 1、2 的响应参数；与当前 **beidou-bridge 代码实现一致**。  
> **状态**：✅ 已写入 `中转系统与北斗系统交互协议0630.docx`；推送 Body 已与代码对齐。

---

## 约定

| 项 | 说明 |
|----|------|
| 中转 → 北斗/调用方 | 成功 **`code: 0`**，`message: "success"` |
| 北斗 → 中转（收推送） | 成功 **`code: 1000`**（协议已定义，不变） |
| HTTP 状态码 | 业务失败时与 Body 中 `code` 一致（如 400/502） |

---

## 一、接口 1 — 注册回调（北斗 → 中转）

**Path**：`POST /api/v1/beidou/callback/register`

**请求 Body（0629，北斗只需 url + frequency）**：

```json
{
  "url": "http://北斗IP:端口/callback",
  "frequency": 4000
}
```

> `vehicleIds` **不在请求中**。中转按本项目配置（`vehicles.yaml` 中 `enabled: true` 的车辆）确定监控名单，在**响应**中返回。

> **幂等**：同一 `url` + `frequency` 可重复 register 以刷新 `vehicles` 快照，**不会**重启定时推送；仅 url 或 frequency 变更时才重建回调任务。

### 1.1 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "registeredAt": 1740469352000,
    "frequency": 4000,
    "vehicleCount": 2,
    "vehicleIds": [
      "hasun-test",
      "LU2606000100"
    ],
    "vehicles": [
      {
        "vehicleId": "hasun-test",
        "online": true,
        "workStatus": 1,
        "battery": 90,
        "position": {
          "x": 114.396,
          "y": 22.704
        },
        "taskId": "dd641d2b",
        "updatedAt": 1740469352000
      },
      {
        "vehicleId": "LU2606000100",
        "online": true,
        "workStatus": 0,
        "battery": 100,
        "position": {
          "x": 114.398,
          "y": 22.702
        },
        "updatedAt": 1740469352000
      }
    ],
    "configChanges": [
      "initial registration"
    ]
  },
  "timestamp": 1740469352000
}
```

| 字段 | 说明 |
|------|------|
| registeredAt | 本次注册生效时间（ms） |
| frequency | 确认的推送周期（ms） |
| vehicleIds | **响应字段**：本项目/本局域网启用的全部巡检车 ID（北斗无需事先知道） |
| vehicles | register 时从云平台查询的快照（可选字段见 RegisterVehicleInfo） |
| configChanges | 相对上次 register 的变更项；**url/frequency 未变重复请求时为 `[]`**（不重启推送） |

### 1.2 失败响应

```json
{
  "code": 400,
  "message": "frequency must be between 3000 and 5000 ms",
  "timestamp": 1740469352000
}
```

| code | HTTP | 含义 |
|------|------|------|
| 400 | 400 | 参数非法（url 空、frequency 超范围、项目未配置可用车辆等） |
| 502 | 502 | 云平台查询失败 |
| 500 | 500 | 中转内部错误 |

## 二、接口 2 — 反控导航（北斗 → 中转）

**Path**：`POST /api/v1/beidou/navigation`

**请求 Body** — 三种等价写法，**只选一种**：

**A. 站名（2010001 taskPoint）**

```json
{
  "vehicleId": "LU2606000100",
  "taskPoint": "中德西北角"
}
```

**B. 坐标（taskPointXYZW，口岸中转发点位）**

```json
{
  "vehicleId": "LU2606000100",
  "taskPointXYZW": { "x": 21.64, "y": 86.28, "z": 0.0, "w": 1.0 }
}
```

**C. 北斗 0630 兼容（中转转 taskPointXYZW）**

```json
{
  "vehicleId": "hasun-test",
  "x": 21.64,
  "y": 86.28,
  "direction": 82,
  "floor": 1
}
```

> `direction`（度）→ `taskPointXYZW.z/w` 四元数；`floor` 暂不下发云侧。

### 2.1 成功响应（同步受理）

已提交云平台即返回成功，**不等待**车到达目标点。

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "vehicleId": "hasun-test",
    "cloudTaskId": "BD-NAV-1740469352001",
    "acceptedAt": 1740469352000
  },
  "timestamp": 1740469352000
}
```

| 字段 | 说明 |
|------|------|
| vehicleId | 请求中的目标车 |
| cloudTaskId | 中转下发云侧任务时使用的 taskId |
| acceptedAt | 受理时间（ms） |

### 2.2 失败响应

```json
{
  "code": 400,
  "message": "invalid or unknown vehicleId",
  "timestamp": 1740469352000
}
```

| code | HTTP | 含义 |
|------|------|------|
| 400 | 400 | 参数缺失或 vehicleId 不在 register/配置中 |
| 404 | 404 | 车辆在云平台不存在（预留） |
| 502 | 502 | 云平台 instructions 调用失败 |
| 500 | 500 | 中转内部错误 |

| 1 | 北斗 POST register（仅 url、frequency） | 响应含 `vehicleIds` + `vehicles` |
