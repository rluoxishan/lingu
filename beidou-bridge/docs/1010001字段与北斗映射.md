# 1010001 字段与北斗推送映射

> **版本**：V1.0（2026-06-30）  
> **协议依据**：《云平台-机器人通信协议-内部标准版.docx》§5.6、§6.1  
> **用途**：现场联调 / 车端确认 / AI 接手时查「推北斗用哪些云字段、枚举是什么」

---

## 1. 数据链路

**有网（实验室）**

```
车端 MQTT 1010001 → 云平台存库 → 中转 HTTP 读取 → 组装 0630 推送 → 北斗
```

**无外网现场（5G 直连，见 [方案变更-5G车端MQTT直连中转.md](./方案变更-5G车端MQTT直连中转.md)）**

```
车端 MQTT 1010001 → 中转机 EMQX → 中转订阅缓存 → 组装 0630 推送 → 北斗
```

中转**不修改** workStatus 枚举值，**不生成**坐标；MQTT/HTTP 读到什么就映射什么。

---

## 2. workStatus → 北斗 state（§5.6.1，原样透传）

| 值 | 枚举名 | 含义 |
|----|--------|------|
| 0 | IDLE | 空闲 |
| 1 | WORKING | 任务中 |
| 2 | FAULT | 故障 |
| 3 | CHARGING | 充电 |
| 4 | EMERGENCY_STOP | 急停 |

代码：`src/mapper/statusMapper.ts` → `state: status.workStatus`

---

## 3. 1010001 → 北斗推送字段对照

| 1010001 / HTTP 字段 | 协议 §6.1 | 北斗推送字段 | 中转处理 |
|---------------------|-----------|--------------|----------|
| `workStatus` | 必填 | `state` | 原样 |
| `battery` | 必填 0～100 | `powerLevel` | 原样 |
| `position_xyz` | 室内车必填，`x,y,z`（米） | `x`, `y` | 取前两段；`positionMode: map_xy` |
| `position` | 室外必填，`经度,纬度,高程` | `x`, `y` | 无 xyz 时作 lon/lat 回退 |
| `heading` | 必填，**度** | `direction` | 四舍五入为 int |
| `taskName` / `taskId` | 任务相关 | `currentTask` | 优先 taskName |
| `errors`（HTTP/tpapi） | 故障 | `isAlert` / `alertList[]` | 有则告警 |
| — | — | `z` | 中转固定 **0**（第三段 yaw 不写入 z） |
| — | — | `floor` | `config/vehicles.yaml` |
| — | — | `vehicleId` | `vehicles.yaml` / register 响应 |

### 中转不使用的 1010001 字段

`robotStatus`、`speedMps`、`vehicleInfo`、`inNodeName`、`nextNodeTime`、`brakeStatus`、`taskProgress`、`isInNode` 等 — **不推北斗**。

### 故障上报

协议规定 **1010004** 为故障状态上报；1010001 内通常无 `errors`。无告警时 `alertList: []` 正常。

## 4. 现场联调需确认的数据（P0）

| # | 字段 | 找谁 | 确认什么 |
|---|------|------|----------|
| 1 | `position_xyz` 或 `position` | **车端** | 定位后是否为 **非 0** 有效值 |
| 2 | `heading` | **车端** | 单位度；有定位时是否更新 |
| 3 | `workStatus` | **车端** | 是否按 §5.6.1 上报 |
| 4 | `battery` | **车端** | 0～100 持续更新 |
| 5 | `taskId` | **车端** | 无任务勿用 `"0"`，用空串 |
| 6 | HTTP 与 MQTT 一致 | **云平台** | admin 查到的与 1010001 同步 |
| 7 | 回调 URL / 网络 | **北斗** | 见 [现场联调检查表.md](./现场联调检查表.md) |

## 5. 示例：LU2606000100（admin HTTP，2026-06-30 商用版）

```
workStatus: 0          → state=0（空闲）
battery: 37             → powerLevel=37
position_xyz: 0.00,0.00,0.00  → x=0, y=0（待定位）
heading: 0              → direction=0
taskId: "0"             → currentTask="0"（建议改为 ""）
taskName: (空)
robotStatus: 1          → 不推北斗
```

| 项 | 解读 |
|----|------|
| workStatus 0 | **IDLE**（空闲），符合 §5.6.1 |
| battery 37 | ✅ 有效 |
| HTTP 含 position_xyz/heading | ✅ 商用版云字段已透出 |
| position_xyz 全 0 | ❌ 待孟泽定位上报 |
| taskId "0" | ⚠️ 无任务建议 `""` |

**推北斗结果（约）**：`state=0, powerLevel=37, x=0, y=0, currentTask="0"`

> 早期 MQTT 样例曾见 workStatus=3（充电）、battery=32；以联调时 **admin HTTP 实时读数** 为准。
