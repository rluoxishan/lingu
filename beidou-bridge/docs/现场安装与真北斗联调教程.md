## 第 2 章：安装 Node.js（中转机若没有）

### 2.1 检查是否已安装

1. 按 `Win + R`，输入 `cmd`，回车。  
2. 输入：

```bat
node -v
```

| 结果 | 下一步 |
|------|--------|
| 显示 `v20.x` 或 `v22.x`、`v24.x` | 跳过本章，进第 3 章 |
| 提示不是内部或外部命令 | 继续 2.2 |

### 2.2 安装 Node.js 20 LTS

1. 浏览器打开：https://nodejs.org/  
2. 下载 **LTS（推荐）** Windows 安装包（.msi）。  
3. 双击安装，**全部默认下一步**即可。  
4. **关闭所有** cmd/PowerShell 窗口，**重新打开** cmd。  
5. 再执行 `node -v`，应显示 v20 及以上。

> 若中转机**完全不能上网**：需提前把 Node.js 安装包也拷进 U 盘。

## 第 7 章：对接真北斗（核心）

### 7.1 发给北斗方的信息（联调前/联调时）

把下面内容发给北斗（可直接转发 `给北斗方-接口一页纸.txt`）：

**① 健康检查（北斗先测能否访问中转）**

```http
GET http://<中转机IP>:8080/health
```

预期：`{"status":"up"}` 或类似 JSON。

**② 注册回调（北斗发起，联调正式开始）**

```http
POST http://<中转机IP>:8080/api/v1/beidou/callback/register
Content-Type: application/json

{
  "url": "<北斗方提供的完整回调URL>",
  "frequency": 4000
}
```

说明：

- **`url`**：北斗提供，中转会按此地址 **每 4 秒 POST 一次** 车辆状态。  
- **`frequency`**：4000 表示 4000 毫秒（4 秒）一轮。  
- **请求里不需要写 vehicleId**；中转按配置返回 **LU2606000100**。

**③ 北斗收到推送后必须回复**

```json
{
  "code": 1000,
  "msg": "成功",
  "timestamp": 1603252608889
}
```

否则中转日志可能出现 `non-success`，推送会被判失败。

**④ 网络要求（三方确认）**

| 方向 | 要求 |
|------|------|
| 北斗 → 中转机:8080 | 能访问 health、register |
| 中转机 → 北斗回调 url | 能 POST 到北斗 |
| 中转机 → sztu.lingubot.cn | 能 HTTPS 拉车辆状态 |

| 已把 register 地址发给北斗 | □ |
| 北斗已提供回调 URL | □ |
| 北斗已测 health 可达 | □ |

### 7.2 你在本机自测 register（可选，北斗未到前）

在 PowerShell 中（**把 URL 换成北斗真实回调**）：

```powershell
$beidouCallback = "http://<北斗IP>:<端口>/callback"
$body = @{ url = $beidouCallback; frequency = 4000 } | ConvertTo-Json
Invoke-RestMethod -Method POST `
  -Uri "http://127.0.0.1:8080/api/v1/beidou/callback/register" `
  -ContentType "application/json" -Body $body | ConvertTo-Json -Depth 8
```

### 7.3 register 成功响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "registeredAt": 1740469352000,
    "frequency": 4000,
    "vehicleCount": 1,
    "vehicleIds": ["LU2606000100"],
    "vehicles": [ { "vehicleId": "LU2606000100", "workStatus": 0, "battery": 85, ... } ],
    "configChanges": ["initial registration"]
  },
  "timestamp": 1740469352000
}
```

**register 当场核对（见第 8 章 R1～R3）。**

### 7.4 推送 Body 示例（北斗每 4s 收到）

```json
{
  "data": {
    "vehicleId": "LU2606000100",
    "x": 21.64,
    "y": 86.28,
    "z": 0,
    "floor": 1,
    "state": 0,
    "powerLevel": 85.0,
    "currentTask": "",
    "isAlert": false,
    "alertList": [],
    "direction": 0
  },
  "timestamp": 1740469352000
}
```

字段以云实际数据为准；**x/y 为 0 不挡 B1 链路验收**。

## 第 9 章：常见问题与处理

| 现象 | 可能原因 | 处理步骤 |
|------|----------|----------|
| 没有 Node.js | 未安装 | 第 2 章安装 LTS，重开窗口 |
| CHECK-env `[FAIL] .env` | 未复制 .env | 从 U 盘复制到根目录 |
| CHECK-env `[FAIL] admin query` 401 | 账号密码错 | 检查 `.env` 三行是否与 admin123 一致 |
| CHECK-env 502 | 云不可达 | ping/浏览器测 `sztu.lingubot.cn` |
| START 报 EADDRINUSE | 8080 被占用 | 任务管理器结束旧 `node.exe`，或改 `config/site/server.yaml` port |
| health 本机 OK，北斗访问不了 | 防火墙 | 管理员运行 OPEN-firewall-8080.bat |
| register code=400 | frequency 不在 3000～5000 | 改为 4000 |
| register code=502 | 云登录失败 | 查 .env；QUERY-vehicle 能否成功 |
| register vehicleIds 空 | vehicles.yaml 无 enabled | 确认 LU2606000100 enabled:true |
| 北斗收不到 POST | url 错 / 中转停了 | 确认 START 窗口开着；北斗 url 可达 |
| 日志 non-success | 北斗未回 code:1000 | 让北斗按协议改响应 JSON |
| x/y 全 0 | 云 position_xyz 空 | 孟泽启商用版；**不挡 B1** |
| navigation 超时 | 云 instructions 慢 | **本次不测 B2** |

**仍无法解决时收集：**

1. START-bridge.bat 窗口**完整日志**（截图）  
2. register 的**请求 Body** 与**响应 JSON**  
3. 北斗侧：是否收到 POST、返回的 HTTP Body  
4. QUERY-vehicle.bat 输出截图
