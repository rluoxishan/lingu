## 1. 你现在要做什么（10 分钟）

在**有网络的开发机**上，进入 `beidou-bridge` 目录：

```powershell
# 1) 复制并填写云平台 admin 账号（勿提交 git）
copy .env.example .env
# 编辑 .env → CLOUD_TENANT_NAME / CLOUD_USERNAME / CLOUD_PASSWORD

# 2) 一键安装 + 检查 + 查云字段
powershell -ExecutionPolicy Bypass -File scripts\pre-site-check.ps1 -Install

# 3) 可选：Mock 北斗全链路预演（会开 3 个窗口）
powershell -ExecutionPolicy Bypass -File scripts\run-pre-site-b1.ps1
```

**通过标准：**

| 步骤 | 预期 |
|------|------|
| `pre-site-check -Install` | 无 `[FAIL]` |
| `query-admin-device` | 能 login；有 `workStatus`、`battery` |
| `run-pre-site-b1` | register 返回 `LU2606000100`；Mock 窗口 ~4s 收到 POST |

`position_xyz` / `heading` 若为 0：**不挡预演**，等孟泽 **IDK 商用版** 启动后再查一次。

## 4. 配置文件说明

### 4.1 `.env`（必填）

```ini
CLOUD_TENANT_NAME=超级管理员
CLOUD_USERNAME=<admin 账号>
CLOUD_PASSWORD=<admin 密码>
```

### 4.2 `config/site/vehicles.yaml`

默认仅 **`LU2606000100`**。现场增加车辆时复制条目并 `enabled: true`：

```yaml
vehicles:
  - vehicleId: "LU2606000100"
    clientId: "LU2606000100"
    floor: 1
    enabled: true
```

### 4.3 与教研 tpapi 的区别

| 用途 | 配置 |
|------|------|
| **现场 B1** | `CONFIG_DIR=./config/site`（`site-start.bat` 已设置） |
| 本地 A 阶段 tpapi | `CONFIG_DIR=./config/tpapi` + `szjsdx` |

**勿混用。**

## 8. 常见问题

| 现象 | 处理 |
|------|------|
| 登录 401 | 检查 `.env` 账号密码 |
| `position_xyz` 全 0 | 孟泽商用版 + 定位；不挡 B1 链路验收 |
| register 502 | 云 login 失败或车辆不在 `vehicles.yaml` |
| 8080 被占用 | `pre-site-check` / `run-pre-site-b1` 会自动尝试释放；或改 `config/site/server.yaml` port |
