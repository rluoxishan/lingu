# 客户电脑部署指南（Windows）

> 在客户 Windows 电脑上安装 beidou-bridge，并设置 **开机自动启动**。

---

## 1. 环境要求

| 项 | 要求 |
|----|------|
| 操作系统 | Windows 10 / 11 或 Windows Server |
| Node.js | **20 或更高**（[下载 LTS](https://nodejs.org/)） |
| 网络 | 能访问云平台 HTTPS；能与北斗内网互通 |
| 磁盘 | 约 200 MB（含 node_modules） |

---

## 2. 安装步骤

### 2.1 获取程序

将 `beidou-bridge` 文件夹复制到客户电脑，建议路径：

```
C:\ProgramData\beidou-bridge
```

或：

```
D:\beidou-bridge
```

**路径不要有中文空格问题**（尽量用英文路径）。

### 2.2 一键安装

以 **管理员** 打开 PowerShell，进入项目目录：

```powershell
cd C:\ProgramData\beidou-bridge
npm run install:windows
```

脚本会：

1. 检查 Node.js
2. 安装依赖 `npm ci`
3. 复制 `.env.example` → `.env`（若不存在）
4. 编译 `npm run build`

### 2.3 填写配置

**① 云平台账号** — 编辑 `.env`：

```ini
CLOUD_TENANT_NAME=超级管理员
CLOUD_USERNAME=实际账号
CLOUD_PASSWORD=实际密码
```

**② 车辆清单** — 编辑 `config\vehicles.yaml`

**③ 服务端口**（可选）— 编辑 `config\server.yaml`，默认 `8080`

---

## 3. 手动测试启动

```powershell
cd C:\ProgramData\beidou-bridge
scripts\windows\start-bridge.bat
```

另开窗口验证：

```powershell
Invoke-RestMethod http://127.0.0.1:8080/health
```

应返回：`status = up`

确认无误后按 `Ctrl+C` 停止。

---

## 4. 注册开机自启动

```powershell
cd C:\ProgramData\beidou-bridge
npm run autostart:install
```

原理：使用 Windows **任务计划程序**，系统启动后 30 秒自动运行；崩溃后自动重试 3 次。

**立即触发一次测试：**

```powershell
Start-ScheduledTask -TaskName BeidouBridge
Invoke-RestMethod http://127.0.0.1:8080/health
```

**查看任务状态：**

```powershell
Get-ScheduledTask -TaskName BeidouBridge
Get-ScheduledTaskInfo -TaskName BeidouBridge
```

**取消开机自启：**

```powershell
npm run autostart:remove
```

---

## 5. 防火墙

若北斗从其他机器访问，需放行入站 **8080**：

```powershell
New-NetFirewallRule -DisplayName "Beidou Bridge" -Direction Inbound -Protocol TCP -LocalPort 8080 -Action Allow
```

## 6. 日志与排错

| 现象 | 处理 |
|------|------|
| 提示找不到 Node.js | 安装 Node.js 20 LTS，重启 PowerShell |
| 提示找不到 dist\main.js | 运行 `npm run build` |
| 登录云失败 | 检查 `.env` 账号密码 |
| 8080 被占用 | 修改 `config\server.yaml` 的 port |
| 自启不生效 | 任务计划程序 → 找 `BeidouBridge` → 查看历史记录 |

任务计划程序路径：`taskschd.msc` → 任务计划程序库 → **BeidouBridge**

---

## 7. 升级程序

```powershell
cd C:\ProgramData\beidou-bridge
git pull          # 或覆盖新文件
npm ci
npm run build
Stop-ScheduledTask -TaskName BeidouBridge -ErrorAction SilentlyContinue
Start-ScheduledTask -TaskName BeidouBridge
```

## 9. Linux 客户（简要）

```bash
cd /opt/beidou-bridge
npm ci && npm run build
cp .env.example .env && nano .env
# systemd 示例见 scripts/linux/beidou-bridge.service
sudo systemctl enable --now beidou-bridge
```
