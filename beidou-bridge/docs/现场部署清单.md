## 2. 第一步：安装（约 10 分钟）

在 **北斗提供的中转机** 上解压部署 zip，例如 `D:\beidou-bridge`：

```powershell
cd D:\beidou-bridge
copy .env.example .env    # 若已有 .env 则跳过
# 编辑 .env 填 CLOUD_*（或直接使用我方带来的 .env）
```

若无 Node.js，先安装 **Node.js 20+**，再运行 `INSTALL.bat` 或：

```powershell
scripts\site-start.bat
```

**有 git 网络时**也可源码安装：

```powershell
git pull origin master
npm run pre-site:install
```

| # | 检查 | 命令 | 预期 | □ |
|---|------|------|------|---|
| I1 | Node 版本 | `node -v` | v20+ | |
| I2 | 编译产物 | `dir dist\main.js` | 文件存在 | |
| I3 | 安装无 FAIL | 上一步输出 | 无 `[FAIL]` | |

**若未安装 Node.js**：见 [客户电脑部署指南.md](./客户电脑部署指南.md) §1，或运行 `npm run install:windows`（管理员 PowerShell）。

## 3. 第二步：配置（约 5 分钟）

### 3.1 云平台账号 `.env`

```powershell
copy .env.example .env
notepad .env
```

填写（示例，以实际账号为准）：

```ini
CLOUD_TENANT_NAME=超级管理员
CLOUD_USERNAME=admin
CLOUD_PASSWORD=admin123
```

> **勿提交 git**。现场用 **admin** 模式，不要用 tpapi 测试号 `szjsdx`。

### 3.2 现场专用配置（一般不用改）

| 路径 | 说明 |
|------|------|
| `config/site/vehicles.yaml` | 已含 **LU2606000100** |
| `config/site/cloud.yaml` | admin + `sztu.lingubot.cn` |
| `config/site/server.yaml` | 监听 **8080** |

增车：编辑 `config/site/vehicles.yaml`，复制条目并 `enabled: true`。

### 3.3 验证云登录与查车

```powershell
npm run query:admin-device
```

| # | 检查 | 预期 | □ |
|---|------|------|---|
| C1 | 脚本能 login | 无 401/502 | |
| C2 | LU2606000100 | 有 `workStatus`、`battery` | |
| C3 | position_xyz / heading | 商用版启动后应非 0；为 0 **不挡 B1** | |

**孟泽职责**：启动 **IDK 商用版**，使车上报有效坐标后再跑一次 `query-admin-device`。

## 4. 第三步：Mock 预演（接北斗前必做，约 15 分钟）

仅监控（中转已运行时）：

## 6. 第五步：接真北斗（B1 读推送）

### 6.1 给北斗方的信息

| 项 | 值 |
|----|-----|
| register 地址 | `POST http://<中转机IP>:8080/api/v1/beidou/callback/register` |
| navigation 地址 | `POST http://<中转机IP>:8080/api/v1/beidou/navigation` |
| health 探活 | `GET http://<中转机IP>:8080/health` |

### 6.2 北斗发起 register

**北斗 → 中转**（只需 `url` + `frequency`）：

```http
POST http://<中转机IP>:8080/api/v1/beidou/callback/register
Content-Type: application/json

{
  "url": "http://<北斗IP>:端口/callback",
  "frequency": 4000
}
```

中转机本机自测（把 URL 换成北斗真实回调）：

```powershell
$body = '{"url":"http://<北斗回调URL>","frequency":4000}'
Invoke-RestMethod -Method POST `
  -Uri "http://127.0.0.1:8080/api/v1/beidou/callback/register" `
  -ContentType "application/json" -Body $body | ConvertTo-Json -Depth 6
```

### 6.3 网络连通（三方确认）

| # | 方向 | 测试 | □ |
|---|------|------|---|
| N1 | 北斗 → 中转机:8080 | 访问 `/health` | |
| N2 | 中转机 → 北斗回调 URL | curl 或北斗确认 | |
| N3 | 中转机 → 云平台 | `npm run query:admin-device` | |

### 6.4 B1 验收（等一个 frequency，约 4s）

| # | 检查 | 预期 | □ |
|---|------|------|---|
| R1 | register HTTP | 200，`code=0` | |
| R2 | vehicleIds | 含 **LU2606000100** | |
| R3 | 北斗收到 POST | 每车每周期 1 次 | |
| R4 | Body | `data.vehicleId` 在 data 内 | |
| R5 | 北斗响应 | **code: 1000** | |
| R6 | 中转日志 | `[scheduler] pushed vehicle=LU2606000100` | |

详细勾选见 [现场联调检查表.md](./现场联调检查表.md) §4。

### 6.5 重启恢复（建议测）

| # | 步骤 | 预期 | □ |
|---|------|------|---|
| T1 | Ctrl+C 停中转 | 推送停止 | |
| T2 | 再运行 `site-start.bat` | 自动恢复推送（读 `data-site/beidou-callback.json`） | |
| T3 | 不重新 register | 北斗仍收到 POST | |

## 7. 第六步：B2 反控（可选，本次一般不验）

> 2026-06-30 站名反控实测：中转收请求 OK，云 `/device/instructions` **10s 超时**。B1 通过后再测。

模拟北斗发站名任务（**慎用，会下发到实车**）：

```powershell
$body = '{"vehicleId":"LU2606000100","taskPoint":"站点名"}'
Invoke-RestMethod -Method POST -Uri http://127.0.0.1:8080/api/v1/beidou/navigation `
```

## 9. 常见问题

| 现象 | 原因 | 处理 |
|------|------|------|
| 登录 401/502 | `.env` 账号错 | 检查 admin 三件套 |
| register 502 | 云 login 失败或车不在清单 | 查 `.env`、`config/site/vehicles.yaml` |
| Mock 9090 起不来 | Windows 系统占用 | 用 `npm run sim:lab`（**19090**） |
| `position_xyz` 全 0 | 商用版未启 / 未定位 | 孟泽启商用版，不挡 B1 |
| 北斗收不到推送 | url 错 / 防火墙 / 中转未启 | 查 N1/N2、8080 |
| scheduler `non-success` | 北斗未回 code 1000 | 北斗改响应格式 |
| navigation 500/超时 | 云 instructions 慢或不可用 | 联系云平台；B2 暂缓 |
| 8080 被占用 | 旧进程 | `pre-site-check` 或手动结束进程 |
