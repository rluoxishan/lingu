# 北斗系统 — 对接需求

> **版本**：V1.1  
> **日期**：2026-06-29  
> **交付对象**：北斗对接团队  
> **用途**：第一期联调需确认与提供的资料；**请北斗方填写 §10 回复栏后回传**  
> **协议依据**：[中转系统与北斗系统交互协议0629.docx](./中转系统与北斗系统交互协议0629.docx)  

---

## 1. 背景

```
车端 ──MQTT──► 云平台 ──HTTP+鉴权──► 中转程序 ──HTTP内网──► 北斗系统
```

- **北斗 ↔ 中转**：HTTP，**内网，无业务鉴权**  
- 第一期严格按协议 **两个接口** 对接  
- 车辆规模：**少量，约 1～10 台**（需在协议基础上确认 **vehicleId** 扩展，见 §4）

### 1.1 双方各做什么

| 方向 | 谁发起 | 能力 |
|------|--------|------|
| 北斗 → 中转 | 北斗 | **接口 1**：注册回调 URL + 推送频率 |
| 中转 → 北斗 | 中转 | **接口 1 续**：按频率 POST 车辆状态到北斗 url |
| 北斗 → 中转 | 北斗 | **接口 2**：反控导航至物理坐标 |
| 中转 → 云 | 中转 | 查状态、下导航（北斗无需提供） |

## 4. 接口 1：注册回调 + 状态推送（P0）

### 4.1 北斗 → 中转（注册）

北斗 POST 至中转（**仅 url + frequency**，无需传 vehicleIds）：

```json
{
  "url": "http://北斗内网地址:端口/回调路径",
  "frequency": 4000
}
```

| 字段 | 说明 |
|------|------|
| url | 所有设备共用同一回调地址 |
| frequency | 推送间隔（毫秒），**3000～5000** |

**请北斗确认：**

| # | 问题 |
|---|------|
| R1 | 注册路径 `POST /api/v1/beidou/callback/register` 是否接受？ |
| R2 | `frequency` **单位确认为毫秒**？ |
| R3 | `frequency` **允许范围 3000～5000 ms** 是否确认？ |
| R4 | 重复注册是否 **覆盖** 旧配置？（中转侧按覆盖实现） |
| R5 | register **响应**返回本项目全部巡检车 `vehicleIds` 及云平台快照，是否可接受？（北斗无需事先知道车 ID） |

**注册成功响应（中转定义）：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "registeredAt": 1740469352000,
    "frequency": 4000,
    "vehicleCount": 2,
    "vehicleIds": ["lingu_test2", "robot-002"],
    "vehicles": [
      {
        "vehicleId": "lingu_test2",
        "online": true,
        "workStatus": 1,
        "battery": 85,
        "position": { "x": 114.398, "y": 22.702 },
        "updatedAt": 1740469352000
      }
    ]
  },
  "timestamp": 1740469352000
}
```

register 时中转向云平台查询**响应中的 vehicleIds**（来自本项目配置）并返回设备信息；之后按 frequency 定时推送，重启后自动恢复，无需重复 register（除非修改 url/frequency 或项目车辆配置变更后需重新 register）。

### 4.2 中转 → 北斗（定时推送）

**协议原文（单车 data）：**

```json
{
  "data": {
    "vehicleId": "lingu_test2",
    "x": 21.64,
    "y": 86.28,
    "z": 1.42,
    "floor": 1,
    "state": 1,
    "powerLevel": 85.0,
    "currentTask": "T001",
    "isAlert": false,
    "alertList": [],
    "direction": 82
  },
  "timestamp": 1740469352000
}
```

有告警时 `isAlert: true`，`alertList` 示例：

```json
"alertList": [{ "alertType": 1, "alertMsg": "故障描述" }]
```

**北斗成功响应：**

```json
{
  "code": 1000,
  "msg": "成功",
  "timestamp": 1603252608889
}
```

**请北斗确认（多车，P0）：**

| # | 问题 | 选项 |
|---|------|------|
| M1 | 1～10 台车如何区分？ | **0629：`data.vehicleId`**，每车独立 POST |
| M2 | 同一 frequency 内多车 | 中转计划 **每车一次 POST** |
| M3 | 某车推送失败 | 中转计划 **互不影响** |

## 5. 接口 2：反控导航（P0）

0629 请求 Body（含 `vehicleId`）：

```json
{
  "vehicleId": "lingu_test2",
  "x": 21.64,
  "y": 86.28,
  "z": 1.42,
  "direction": 82,
  "floor": 1
}
```

**请北斗确认：**

| # | 问题 |
|---|------|
| N1 | 是否 **同意增加 `vehicleId` 字段**？ |
| N2 | `vehicleId` 由谁分配、与北斗终端号是否有一一映射？ |
| N3 | 反控路径 `POST /api/v1/beidou/navigation` 是否接受？ |
| N4 | 响应是否接受成功 `code: 0`（中转定义，与北斗回调 `1000` 区分）？ |
| N5 | 反控是 **同步受理**（已下发即成功）还是需等车到达？（第一期按 **同步受理**） |

**反控成功响应（中转定义）：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "vehicleId": "lingu_test2",
    "cloudTaskId": "BD-NAV-1740469352001",
    "acceptedAt": 1740469352000
  },
  "timestamp": 1740469352000
}
```

## 6. 字段语义确认（P0）

| 字段 | 协议注释 | 需确认 |
|------|----------|--------|
| **x, y** | 物理 x/y 坐标 | 单位：**米**？还是 **经纬度**？与云 `position_xyz` 是否同一系？ |
| **z** | 物理 z 坐标 | **高程（米）** 还是其他？云侧第三段常为 yaw 弧度 |
| **floor** | 楼层 | 取值范围、默认层、是否必填 |
| **state** | 车辆状态 | 是否与 0 空闲 / 1 任务中 / 2 故障 / 3 充电 / 4 急停 一致？ |
| **powerLevel** | 电量 % | 0～100 double |
| **currentTask** | 目前任务 | taskId 还是任务名称？无任务时空串？ |
| **isAlert** | 是否需要告警 | true 的判定规则 |
| **alertList** | 告警集合 | 无告警 `[]`；每项含 `alertType`、`alertMsg`（见 §6.1） |
| **direction** | 车辆方向 | **单位：度？** 范围 0～360？ |
| **timestamp** | 时间戳 | **毫秒** 还是秒？ |

### 6.1 alertList[].alertType 枚举（请北斗填写）

| alertType | 含义 |
|-----------|------|
| 0 | （如：无告警） |
| 1 | |
| … | |

### 6.2 state 枚举（中转计划映射云 workStatus）

| state | 含义 |
|-------|------|
| 0 | 空闲 |
| 1 | 任务中 |
| 2 | 故障 |
| 3 | 充电 |

## 7. 错误处理与联调约定（P1）

| # | 问题 | 中转默认行为 |
|---|------|--------------|
| E1 | 北斗返回 `code != 1000` | 记日志，有限重试 2 次 |
| E2 | 北斗回调 url 超时 | 中转默认 5s |
| E3 | Body 字段缺失 | 请说明北斗返回什么 code |
| E4 | 联调测试数据 | 是否提供标准测试坐标、期望 state |
