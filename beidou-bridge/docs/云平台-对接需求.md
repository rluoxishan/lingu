# 云平台 — 对接需求

> **版本**：V1.1  
> **日期**：2026-06-29  
> **交付对象**：云平台 / 后端开发团队  
> **用途**：第一期中转程序对接所需的接口文档、字段与业务规则；**请云平台填写 §8 回复栏后回传**  

---

## 1. 背景

```
车端 ──MQTT──► 云平台 ──HTTP+鉴权──► 中转程序 ──HTTP内网──► 北斗系统
```

中转程序 **不直连 EMQX / 车端 MQTT**，仅通过云平台 HTTP API：

1. **读**：按车查询最新状态 → 定时推送给北斗  
2. **写**：接收北斗反控坐标 → 向指定车辆下发导航任务（2010001）

**第一期规模**：约 **1～10 台车**；状态轮询约 **1Hz/车**（10 车 ≈ 10 QPS 读状态 + 偶发写导航）。

#### 功能 D1：tpapi 下发任务点位（双模式，对齐 2010001）

| 文档项 | 说明 |
|--------|------|
| **路径** | `POST .../tpapi/device/set_device_task_point` |
| **鉴权** | `Authorization: Bearer {tpapi accessToken}` |
| **原则** | **二选一**：`taskPoint`（任务名）或 `taskPointXYZW`（坐标）；另一项为空 |

**模式 A — 任务名（2010001 站名，`taskNodes[].taskPoint`）**

```json
{
  "deviceId": "LU2606000100",
  "taskPoint": "中德西北角",
  "taskPointXYZW": null
}
```

**模式 B — 目标点坐标（口岸中转发点位，2010001 坐标节点 x/y/z/w）**

```json
{
  "deviceId": "LU2606000100",
  "taskPoint": "",
  "taskPointXYZW": {
    "x": 21.64,
    "y": 86.28,
    "z": 0.0,
    "w": 1.0
  }
}
```

| 字段 | 含义 |
|------|------|
| `taskPoint` | 站点/任务名；坐标模式下传 **空串** |
| `taskPointXYZW.x/y` | 地图坐标（米，与 2010001 一致） |
| `taskPointXYZW.z/w` | 姿态四元数分量（2010001 节点 z/w，非物理高度） |

**中转映射**

| 来源 | 云 Body |
|------|---------|
| navigation `{ vehicleId, taskPoint }` | 模式 A |
| navigation `{ vehicleId, taskPointXYZW }` | 模式 B |
| navigation `{ vehicleId, x, y, direction }`（北斗 0630） | 模式 B（direction 度 → z/w 四元数） |
| admin `apiMode` + 坐标 | `POST /device/instructions` type=2010001（同语义） |

**冒烟**

```powershell
npm run test:tpapi-set-task-point -- lingu_test2 中德西北角
npm run test:tpapi-set-task-point -- lingu_test2 --xyzw 21.64 86.28 0.0 1.0
```
