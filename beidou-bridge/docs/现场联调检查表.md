## 4. B1 现场读链路：北斗 register + 推送

### 4.1 北斗发起 register

**请求**（北斗 → 中转，**只需 url + frequency**）：

```http
POST http://<中转IP>:8080/api/v1/beidou/callback/register
Content-Type: application/json

{
  "url": "http://<北斗IP>:端口/callback",
  "frequency": 4000
}
```

PowerShell 自测（在中转本机）：

```powershell
$body = '{"url":"http://<北斗回调URL>","frequency":4000}'
Invoke-RestMethod -Method POST `
  -Uri "http://127.0.0.1:8080/api/v1/beidou/callback/register" `
  -ContentType "application/json" -Body $body | ConvertTo-Json -Depth 6
```

### 4.2 register 验收

| # | 检查项 | 预期 | □通过 | 备注 |
|---|--------|------|-------|------|
| R1 | HTTP 状态码 | 200 | | |
| R2 | `code` | 0 | | |
| R3 | `data.vehicleIds` | 与 `vehicles.yaml` 启用列表一致 | | **北斗从响应获知车 ID** |
| R4 | `data.vehicles` | 有云快照（battery、workStatus 等） | | |
| R5 | `configChanges` | 重复且 url/frequency 未变时为 `[]` | | 不重启推送 |
| R6 | 重复 register | 推送不中断，vehicles 为最新云快照 | | 幂等刷新 |

### 4.3 等待一个 frequency 周期（如 4s）

| # | 检查项 | 预期 | □通过 | 备注 |
|---|--------|------|-------|------|
| P1 | 北斗收到 POST 次数 | N 台车 ≈ N 次/周期 | | **每车单独 POST** |
| P2 | Body 结构 | `data.vehicleId` 在 data 内 | | |
| P3 | 告警字段 | `alertList: []` 或 `[{alertType,alertMsg}]` | | |
| P4 | 北斗 HTTP 响应 | `code: 1000` | | |
| P5 | 中转日志 | `[scheduler] pushed vehicle=...` | | |

**推送 Body 参考**（见 [协议0629-中转响应定稿.md](./协议0629-中转响应定稿.md) §四）。

### 4.4 重启恢复（建议测）

| # | 步骤 | 预期 | □通过 |
|---|------|------|-------|
| T1 | 停止中转进程 | 推送停止 | |

## 6. 常见问题速查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| register 400 `no enabled vehicles` | `vehicles.yaml` 无 `enabled: true` | 编辑 yaml 后重新 register |
| register 502 | 云登录失败或 deviceId 不存在 | 查 `.env`、账号绑定车辆 |
| 北斗收不到推送 | url 错、防火墙、中转未启动 | 查 N1/N2、回调 URL |
| 推送有但坐标全 0 | 云 `position_xyz` 为空 | 找云平台/孟泽；非中转 bug |
| scheduler 报 non-success | 北斗未返回 code 1000 | 北斗改响应格式 |
| navigation 502 | 云 2010001 不可用 | 等云平台接口 |
