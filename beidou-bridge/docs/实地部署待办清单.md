## 6. 阶段 D：部署上线

### 6.1 部署前准备

| # | 事项 | 说明 |
|---|------|------|
| D1 | 服务器 | Node.js ≥ 20；与北斗同网段或可内网互访 |
| D2 | 代码部署 | `git clone` → `npm ci` → 配置 `config/` |
| D3 | 环境变量 | `CLOUD_TENANT_NAME`、`CLOUD_USERNAME`、`CLOUD_PASSWORD`（勿入 git） |
| D4 | 车辆配置 | 编辑 `config/vehicles.yaml` |
| D5 | 云配置 | `config/cloud.yaml` 设 `mock: false` |
| D6 | 防火墙 | 北斗 → 中转 `:8080`；中转 → 北斗回调 URL；中转 → 云 HTTPS 出站 |
| D7 | 进程守护 | systemd / PM2 / Windows 服务，崩溃自动重启 |

### 6.2 启动命令

```powershell
cd beidou-bridge
$env:CLOUD_TENANT_NAME = "超级管理员"
$env:CLOUD_USERNAME = "bridge-bot"
$env:CLOUD_PASSWORD = "your-password"
npm start
```

默认监听 `http://0.0.0.0:8080`。

### 6.3 上线步骤（按序）

```
① 部署并启动 → GET /health 返回 {"status":"up"}
② 确认 vehicles.yaml 与云 deviceId 一致
③ 确认云账号登录正常
④ 北斗 POST /api/v1/beidou/callback/register 注册回调
⑤ 观察北斗收到定时推送
⑥ 北斗 POST /navigation 测反控
⑦ 重启服务，确认注册恢复、推送继续
```

### 6.4 接入关系速查

| 方向 | 谁主动 | 做什么 |
|------|--------|--------|
| 北斗 → 中转 | 北斗 | 注册回调 URL、反控导航 |
| 中转 → 北斗 | 中转 | 按 frequency POST 车辆状态 |
| 中转 → 云 | 中转 | 登录、查状态、下发 2010001 |
| 云 → 中转 | 无 | 云不需回调中转 |

**阶段 D 完成标志：** 生产/准生产环境稳定运行；运维文档交付。
