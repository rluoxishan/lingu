# 模拟联调实验室（Mock 北斗 + 真云）

> **目的**：真北斗对接前，用 **模拟北斗** 验证两条链路是否正常。  
> **真云**：`sztu.lingubot.cn` admin + 现场车 `LU2606000100`  
> **通过后再**：部署到中转机 → register 的 `url` 改为北斗回调地址

---

## 1. 两条链路

```
读（B1）:  车端 ──1010001──► 云平台 ──HTTP──► 中转 ──POST──► Mock北斗(19090)
写（B2）:  Mock北斗 ──navigation──► 中转 ──HTTP──► 云平台 ──► 车端
```

| 链路 | 验证什么 | 通过标志 |
|------|----------|----------|
| **读** | 云字段 → 中转映射 → Mock 收推送 | Mock 窗口 ~4s 有 POST；monitor **OK** |
| **写** | Mock 发任务 → 云受理 | navigation 返回 `code:0`（2026-06-30 站名实测云 instructions 超时） |

> **端口说明**：Mock 默认 **19090**。本机 **9090** 常被 Windows 系统占用（EACCES），勿用。

---

## 2. 一键启动

```powershell
cd beidou-bridge
copy .env.example .env   # 填 CLOUD_*（现场 admin）

# 真云 + LU2606000100（需有效 .env）
npm run sim:lab

# .env 占位时：Mock 云 + lingu_test2（仅验读链路逻辑）
npm run sim:lab:mock

# 仅监控
npm run sim:monitor

# 写链路（站名，慎用实车）
npm run sim:write -- -TaskPoint "中德西北角"
```

中文站名 POST 建议 UTF-8：

```powershell
$body = '{"vehicleId":"LU2606000100","taskPoint":"中德西北角"}'
Invoke-RestMethod -Method POST -Uri http://127.0.0.1:8080/api/v1/beidou/navigation `
  -ContentType "application/json; charset=utf-8" `
  -Body ([System.Text.Encoding]::UTF8.GetBytes($body))
```

## 3. 脚本

| 脚本 | 作用 |
|------|------|
| `scripts/sim/mock-beidou-server.mjs` | **推荐** Mock 北斗（Node，19090，回 code:1000） |

| `scripts/sim/read-path-monitor.ps1` | 云 HTTP vs Mock 推送，OK/MISMATCH |
| `scripts/sim/write-path-test.ps1` | 模拟北斗 POST navigation |
| `scripts/run-simulation-lab.ps1` | 一键：Mock + 中转 + register + monitor |

---

## 4. 与真北斗对接

| 阶段 | Mock | 真北斗 |
|------|------|--------|
| 远程开发 | ✅ `npm run sim:lab` | 不需要 |
| 现场部署 | 先 Mock 再换 url | register `url` = 北斗回调 |
| B2 反控 | `sim:write` | 北斗 POST navigation |

**唯一替换点**：register 里 `url` 从 `http://127.0.0.1:19090/callback` → 北斗提供地址。
