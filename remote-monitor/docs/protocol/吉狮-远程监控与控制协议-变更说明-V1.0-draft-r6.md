## 二、已确认并写入协议的改动

### 2.1 架构与设备分工 ✅

- **摄像机为独立设备**，不受车辆底盘 MQTT 控制。
- **MCU 串口 A1/A2**（830.pdf）：速度/转向/急停/大灯/转向灯/超声/屏幕；**A2 仅 6 字节，无作业/刹车灯/喇叭**。
- **中位机 UDP**：视频推流、云台、作业机构、位姿扩展。
- **吉狮监控页开流**：一次 HTTP → 后台调厂商协议 → 响应 **playUrl** → WebRTC；**前端不涉及 2010004**。

### 2.2 视频 / 推流 ✅（Hasun 口径）

| 项 | 结论 |
| -- | ---- |
| 推流协议 | **WebRTC** |
| 推流地址 | **云平台后台分配** |
| 监控页开流 | **云平台 HTTP 一次请求** |
| 播放 | 同次响应 **playUrl**，前端 WebRTC |
| 厂商 HTTP | 后台内部，前端不看 |
| 2010004 | **MQTT 直连方**仍有效；与监控页 HTTP **并行存在**，非二选一 |

### 2.3 2010005 远程驾驶 ✅（瑜权清单）

| 项 | 结论 |
| -- | ---- |
| 模式 | **A+B 并存**：有 `command` 走模式 A，否则走 `speed`/`angle`（模式 B，**speed=m/s、angle=rad**，车型兼容在车端） |
| 模式 A 枚举 | FORWARD / BACKWARD / LEFT / RIGHT / STOP / PAUSE / EMERGENCY_STOP |
| speedLevel | 1–3；前进 0.3/0.6/1.0 m/s，后退 0.15/0.3/0.5 m/s（联调可微调） |
| seq | 单调递增，防乱序 |
| 频率 | 10–30 Hz，松开发 STOP；不超过 30 Hz，不低于 10 Hz |
| 档位 P/N/R/D | P/N→STOP；R→BACKWARD L1；D→FORWARD L1 |
| 云台 | **不推荐** 2010005 内嵌 cameraPan；走 **2010002 GIMBAL** |

### 2.4 2010008 急停 ✅

- `EMERGENCY_STOP` / `RELEASE`；QoS 2、有 reply。
- **监控页急停按钮走 2010008**（不用独立「刹车灯」）。

### 2.5 2010002 设备控制 ✅

**flat（MCU · 部分已有）**

| 字段 | 状态 |
| ---- | ---- |
| turnSignals 0–3 | ✅ 已实现 |
| headlights 0/1 | ✅ 已实现 |
| headlights 2/3 远光/全开 | ⚠️ MCU 支持，**代码待放开** |
| ultrasonic / screen / brake / remoteControl | ✅ 已实现 |

**target 模型（中位机 · 协议已定）**

| target | command / params | 用途 |
| ------ | ---------------- | ---- |
| WORK | SET + SWEEP/SPRAY/SUCTION/DUMP | 清扫/洒水/吸污/卸料 |
| GIMBAL | MOVE / MOVE_STEP / ZOOM_IN/OUT / HOME / PRESET / STOP | 云台 |
| AUDIO | SET + HORN + duration | 喇叭 |
