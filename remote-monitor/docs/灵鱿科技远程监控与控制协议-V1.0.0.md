## 1 编写目的

定义 **远程监控与控制模块** 中，第三方平台与 **车端** 之间的 MQTT 通信规范，供第三方平台统一引用。

V1.0 覆盖两类能力：

1. **车端摄像头推流**（2010004）
2. **车端远程控制**（2010002 / 2010005 / 2010008）

通讯链路：

```
第三方平台 ── MQTT（dev/sub · dev/reply）──► EMQX ──► 车端 MQTT 适配层 ──► 车端业务软件
```

车辆周期性状态（1010001）、任务下发、地图导航等能力 **不在本文档范围**，见《云平台-机器人通信协议》全量文档。

## 3 系统架构

```
第三方监控/控制平台 ──MQTT──► EMQX ──MQTT──► 车端 MQTT 适配层 ──► 车端业务软件
         │                              │
         │  发布 dev/sub/{clientId}      │  订阅 dev/sub
         │  订阅 dev/reply/{clientId}    │  发布 dev/reply
         └──────────────────────────────┘
```

**对接原则：**

1. 第三方以 **MQTT 客户端** 身份连接 EMQX，与车端使用同一套 Topic 规范。
2. 车辆唯一标识为 **`clientId`**（与 MQTT ClientId 一致）。
3. 密码/密钥解析规则由部署方私下约定，不在本文档公开。

## 4 连接与认证

### 4.1 车辆注册

1. 在平台注册车辆，获得 `clientId` 及密钥文件  
2. 密钥文件部署至车端指定路径  
3. 配置 EMQX 服务地址、端口  

### 4.2 MQTT 连接参数

| 参数 | 值 | 说明 |
| ---- | -- | ---- |
| ClientId | 车辆注册编号 | 与 `clientId` 一致 |
| Username | 车辆注册编号 | 同 ClientId |
| Password | 自密钥文件解析 | 解析规则私下约定 |
| Keep Alive | 60 秒（推荐） | — |
| 协议版本 | MQTT 3.1.1 / 5.0 | 以 EMQX 部署为准 |

### 4.3 在线状态（可选）

EMQX 规则引擎将上下线事件转发至 `online_status/{clientId}`（retain）。第三方可订阅 `$share/{app_name}/online_status/+` 获取车队在线状态。

| ----- | ---- |

### 5.3 车端行为

| Topic | 方向 | 说明 |
| ----- | ---- | ---- |
| `dev/sub/{clientId}` | 车端订阅 | 接收控制指令 |
| `dev/reply/{clientId}` | 车端发布 | 回复指令结果 |

## 6 消息信封

### 6.1 下发指令（第三方 → 车）

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| id | Long | 是 | 消息唯一 ID（建议雪花算法），车端回复时原样回传 |
| time | Long | 是 | 毫秒时间戳 |
| type | String | 是 | 指令类型，格式 `AAABBBB` |
| data | Object | 是 | 业务数据体 |

```json
{
  "id": 1686004858937364480,
  "time": 1740469352000,
  "type": "2010005",
  "data": {}
}
```

### 6.2 指令回复（车 → 第三方）

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| id | Long | 回传下发的 id |
| code | Integer | 200=成功，500=失败 |
| msg | String | 描述 |
| time | Long | 毫秒时间戳 |
| type | String | 回传下发的 type |
| data | Object | 附加数据 |

```json
{
  "id": 1686004858937364480,
  "code": 200,
  "msg": "ok",
  "time": 1740469352100,
  "type": "2010004",
  "data": {}
}
```

### 6.3 通用约定

| 约定 | 说明 |
| ---- | ---- |
| 未知字段 | 接收方忽略，不报错 |
| 回复超时 | 除 **2010005** 外，车端应在 **5 秒内** dev/reply |
| QoS | 见各指令章节；2010005 高频场景可降为 0 |
| clientId | 统一用 MQTT ClientId，payload 中 **不得重复传 vehicleId** |

## 8 远程控制

远程控制分为三类：**底盘遥控**（2010005）、**设备控制**（2010002）、**急停**（2010008）。

### 8.1 2010005 遥控车辆

#### 8.1.1 说明

远程遥控底盘运动；可与云台角/变焦在同一报文中组合下发。  
**车端不要求 dev/reply 回复**（低时延优先）。

| 项目 | 值 |
| ---- | -- |
| Topic | `dev/sub/{clientId}` |
| QoS | 2（高频遥控可协商降为 0） |
| 车辆回复 | **不需要** |

#### 8.1.2 data 字段

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| speed | Integer | 否 | 底盘速度（IOT 兼容字段） |
| angle | Integer | 否 | 底盘转向角（IOT 兼容字段） |
| command | String | 否 | 遥控命令枚举，见 §10.4 |
| speedLevel | Integer | 否 | 速度档位 1–N |
| cameraPan | Number | 否 | 云台水平角 |
| cameraTilt | Number | 否 | 云台俯仰角 |
| cameraZoom | Number | 否 | 变焦倍数 |
| seq | Integer | 否 | 指令序号，防乱序 |

**两种兼容模式（优先 command）：**

| 模式 | 字段 | 适用 |
| ---- | ---- | ---- |
| A. command | `command` + `speedLevel` + `seq` | 方向键、档位 |
| B. speed/angle | `speed` + `angle` | IOT 兼容 |

**模式 A — command 与 speedLevel**

| command | 说明 | speedLevel 对应速度 |
| ------- | ---- | ----------------- |
| FORWARD | 前进 | L1/L2/L3 → 0.3 / 0.6 / 1.0 m/s |
| BACKWARD | 后退 | L1/L2/L3 → 0.15 / 0.3 / 0.5 m/s |
| LEFT / RIGHT | 原地转向 | ±0.3 rad，忽略 speedLevel |
| STOP | 停止 | speed=0, angle=0 |
| PAUSE | 暂停 | 预留 |

**档位 P/N/R/D 映射：** P/N→STOP；R→BACKWARD+L1；D→FORWARD+L1。

#### 8.1.3 示例：远程前进

```json
{
  "id": 1686004858937364485,
  "time": 1740469352000,
  "type": "2010005",
  "data": {
    "command": "FORWARD",
    "speedLevel": 2,
    "seq": 1001
  }
}
```

#### 8.1.4 示例：IOT 兼容

```json
{
  "id": 1686004858937364486,
  "time": 1740469352100,
  "type": "2010005",
  "data": {
    "speed": 50,
    "angle": 0
  }
}
```

#### 8.1.5 建议下发频率

| 场景 | 建议频率 | 说明 |
| ---- | -------- | ---- |
| 远程驾驶 | 5–10 Hz | 松开按键发 `STOP` |
| 云台微调 | 事件触发 | 也可用 2010002 GIMBAL |
| 急停 | 立即单次 | 优先使用 2010008 |

### 8.2 2010002 修改参数 / 设备控制

#### 8.2.1 说明

用于 **灯光、转向、制动、云台、作业机构** 等离散控制。  
支持 **flat 字段** 与 **target 模型** 两种方式，可并存。

| 项目 | 值 |
| ---- | ---- |
| Topic | `dev/sub/{clientId}` |
| QoS | 2 |
| 车辆回复 | **需要**，5 秒内 dev/reply |

#### 8.2.2 flat 字段（灯光 / 转向）

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| brake | Integer | 制动，见 §10.1 |
| headlights | Integer | 大灯 0/1/2/3（关/近光/远光/全开） |
| ultrasonic | Integer | 超声波 0/1 |
| screen | Integer | 屏幕 0/1 |
| turnSignals | Integer | 转向灯，见 §10.2 |

**示例：远光 + 双闪**

```json
{
  "id": 1686004858937364480,
  "time": 1740469352000,
  "type": "2010002",
  "data": {
    "headlights": 1,
    "turnSignals": 3
  }
}
```

#### 8.2.3 target 设备控制（云台 / 作业 / 喇叭）

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| target | String | GIMBAL / WORK / AUDIO |
| command | String | 见下表 |
| params | Object | 必填；无参时传 `{}` |

**target 枚举**

| target | 说明 |
| ------ | ---- |
| GIMBAL | 云台 PTZ |
| WORK | 作业机构 |
| AUDIO | 喇叭 |

**GIMBAL command：** MOVE、MOVE_STEP、ZOOM_IN、ZOOM_OUT、HOME、SET_PRESET、GOTO_PRESET、STOP  

**WORK command：** SET（params.action 为作业类型，on=true/false）、STOP

**示例：云台步进右转**

```json
{
  "type": "2010002",
  "data": {
    "target": "GIMBAL",
    "command": "MOVE_STEP",
    "params": { "pan": 5.0 }
  }
}
```

**示例：启动作业**

```json
{
  "type": "2010002",
  "data": {
    "target": "WORK",
    "command": "SET",
    "params": { "action": "SWEEP", "on": true }
  }
}
```

#### 8.2.4 典型控制映射

| 操作 | MQTT |
| ---- | ---- |
| 前进/后退/转向/停止 | 2010005 command + seq |
| 档位 P/N/R/D | 2010005，见 §8.1.2 |
| 急停 / 解除 | 2010008 EMERGENCY_STOP / RELEASE |
| 近光/远光/转向/双闪 | 2010002 headlights / turnSignals |
| 作业机构 | 2010002 target=WORK |
| 喇叭 | 2010002 target=AUDIO |
| 云台 | 2010002 target=GIMBAL |

## 9 摄像头推流：2010004

### 9.1 说明

控制车端 **摄像机** 向指定地址推流。推流协议为 **WebRTC**；推流地址由 **流媒体服务** 分配后，经 2010004 下发至车端。

开启推流后，须 **周期重复下发** 2010004 作为心跳；若在约定周期内未收到，车端 **自动关闭推流**。

| 项目 | 值 |
| ---- | -- |
| Topic | `dev/sub/{clientId}` |
| QoS | 2 |
| 车辆回复 | **需要** |
| 传输协议 | WebRTC |
| url 来源 | 流媒体服务分配 |

### 9.2 data 字段

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| status | Integer | 是 | 0=关闭，1=开启 |
| view | Integer | 条件 | 摄像头视角，见 §9.3 |
| url | String | 条件 | WebRTC 推流地址；status=1 时必填 |

`url` 具体格式（WHIP/WHEP、信令 URL、token 等）由流媒体服务定义。

### 9.3 摄像头视角（view）

| 值 | 枚举名 | 说明 |
| -- | ------ | ---- |
| 1 | FRONT | 前视 |
| 2 | RIGHT | 右视 |
| 3 | REAR | 后视 |
| 4 | LEFT | 左视 |

环视（surround）、云台（PTZ）是否扩展 view=5/6：**待确认**。

### 9.4 推流约定

| 项 | 约定 |
| -- | ---- |
| 车端推流协议 | WebRTC |
| 推流 url | 流媒体服务分配，经 2010004 下发车端 |
| 编码 / 分辨率 / 码率 | 流媒体服务默认或配置 |

### 9.5 心跳关流

开启推流后须以一定周期重复下发 2010004；若连续 n 个周期未收到，则关闭推流。

| 参数 | 建议值 | 说明 |
| ---- | ------ | ---- |
| heartbeatIntervalSec | 5 | 重复下发开流指令的间隔（秒） |
| maxMissedHeartbeats | 3 | 容忍丢失次数，超时关流 |

### 9.6 单路示例

```json
{
  "id": 1686004858937364484,
  "time": 1740469352000,
  "type": "2010004",
  "data": {
    "status": 1,
    "view": 1,
    "url": "https://stream.example.com/webrtc/push/{sessionId}"
  }
}
```

### 9.7 多路视频

每路摄像头 **独立申请推流 url**，再 **独立下发 2010004**（各自 view + url + 心跳）。

```
分配 url → 2010004 { view:1, url: "..." }  // 前视
分配 url → 2010004 { view:2, url: "..." }  // 右视
…
```

### 9.8 关闭推流

```json
{
  "id": 1686004858937364487,
  "time": 1740469353000,
  "type": "2010004",
  "data": {
    "status": 0,
    "view": 1
  }
}
```

## 11 典型联调流程

### 11.1 打开单路视频

```
1. 第三方准备流媒体服务，获得 push URL
2. 每 5s 发布 2010004 { status:1, view:1, url:"..." }
3. 等待 dev/reply code=200
4. 在流媒体侧拉流验证
5. 停止时发 status:0 或停止心跳
```

### 11.2 远程驾驶

```
1. 开启所需视频路 2010004
2. 以 5–10Hz 发布 2010005 { command:"FORWARD", speedLevel:2, seq:N }
3. 松开时发 command:"STOP"
4. 急停：2010008 EMERGENCY_STOP（优先于 2010005）
```

### 11.3 云台控制

```
2010002 { target:"GIMBAL", command:"MOVE_STEP", params:{ pan, tilt } }
```

## 13 与完整协议及后续版本的关系

| 文档 / 版本 | 关系 |
| ----------- | ---- |
| 《云平台-机器人通信协议》（全量版） | 母版；含 1010001 遥测、2010001 任务、地图、RPC 等 |
| **本文档 V1.0** | 远程监控与控制模块之 **MQTT 车端篇**：推流 + 远程控制 |
| **V2 云平台 API 篇**（规划） | 补充灵鱿云平台 HTTP OpenAPI；与 V1.0 组合为完整远程监控与控制模块 |

**模块组成示意：**

```
完整远程监控与控制模块
├── V1.0 本文档（MQTT 车端）
│     ├── 2010004 摄像头推流
│     └── 2010002 / 2010005 / 2010008 远程控制
└── V2 云平台 API（规划）
```

1010001 周期性状态上报、任务进度等 **不属于** 远程监控与控制模块 V1.0。第三方若需展示车辆状态，应通过 V2 云平台 API 或全量 MQTT 协议获取。
