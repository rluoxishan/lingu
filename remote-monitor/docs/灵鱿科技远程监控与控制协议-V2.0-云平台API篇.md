## 1 编写目的

定义第三方监控/控制平台通过 **HTTP/HTTPS** 对接灵鱿云平台的接口规范，包括：

1. 会话鉴权
2. 车辆状态查询（聚合车端 **1010001**）
3. 摄像头开流/关流（WebRTC playUrl，经 **2010004** 转发）
4. 远程控制指令下发（转发至车端 MQTT）

> **室内雷达感知**（2010012 / map_meta / 1010003）不在本文范围，见 `remote-monitor/docs/internal/监控页-雷达感知-云平台API契约.md` 及 Word 确认单。

**协议关系**：本文描述 **admin-api HTTP/WS**；车端 MQTT 载荷语义以《云平台-机器人通信协议-内部标准版》为准。HTTP 层 `code: 0` 表示云平台受理；MQTT `dev/reply` 使用 `code: 200`，二者不混用。

**Base URL**：`https://sztu.lingubot.cn/admin-api`（sztu 联调示例；生产以实际部署为准）。我方无独立测试环境，前端本地 `npm run dev` 直连上述 Base URL 抓包验收。

**通用请求头**

| 头 | 说明 |
| -- | ---- |
| `Authorization` | `Bearer {accessToken}` |
| `Content-Type` | `application/json` |

**通用响应结构**

```json
{
  "code": 0,
  "msg": "success",
  "data": {}
}
```

| code | 说明 |
| ---- | ---- |
| 0 | 成功 |
| 401 | 未登录或 Token 失效 |
| 403 | 无权限 |
| 500 | 服务器内部错误 |
| 502 | 车端离线或指令转发失败 |

## 4 摄像头开流（2010004）

云平台通过统一的 **`POST /device/instructions`** 下发 **type = 2010004**，内部完成推流调度并返回 WebRTC **播放信令地址**。前端对每路 `url` 发起信令交换后即可播放。

### 4.1 开启推流

| 项 | 值 |
| -- | -- |
| 路径 | `POST /device/instructions` |
| Body | 见下 |

**请求示例**（sztu 测试环境 `deviceId=hasun-test` 已验证）

```json
{
  "deviceId": "hasun-test",
  "type": "2010004",
  "data": {
    "status": 1,
    "view": 1
  }
}
```

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| deviceId | String | 是 | 车辆 ID |
| type | String | 是 | 固定 `2010004` |
| data.status | Integer | 是 | `1` = 开启推流 |
| data.view | Integer | 是 | 摄像头视角，见下表；每次请求开启 **指定一路**，响应 `data` 通常为 **0 或 1 条** |

**view 枚举**

| view | 说明 | 响应 info 示例 |
| ---- | ---- | -------------- |
| 1 | 前视 | 前 |
| 2 | 右视 | 右 |
| 3 | 后视 | 后 |
| 4 | 左视 | 左 |
| 5 | 环视（五路监控第 5 路） | 环视（**待 Hasun 确认**确切文案） |
| 6 | 云台（云台控制模式，可选） | 云台（**待确认**） |

**五路远程监控**：对 `view` = **1、2、3、4、5** 分别请求（可并行），合并 `data[]` 后播放。

**单路请求示例**（`view=4` 开启左视，sztu 已验证）

```json
{
  "deviceId": "hasun-test",
  "type": "2010004",
  "data": {
    "status": 1,
    "view": 4
  }
}
```

**单路响应示例**

```json

  "data": [
    {
      "name": "KRIPC_93002892_28",
      "url": "https://sztu-video.lingubot.cn/index/api/webrtc?app=live&stream=camera/KRIPC_93002892_28&type=play",

      "status": 1
```

### 4.2 WebRTC 播放信令

对 §4.1 返回的每路 `url` 执行 **POST**（ZLMediaKit WebRTC API）。

**请求 Body**

```json
{
  "sdp": "<浏览器 offer SDP>",
  "type": "offer"
}
```

**响应示例**

```json
{
  "code": 0,
  "id": "rBi/Hx9AH0A=_59",
  "sdp": "v=0\r\no=- ...",
  "type": "answer"
}
```

前端将 `answer` SDP 设为 `RTCPeerConnection` 远端描述后即可收流播放。流媒体服务域名为 `sztu-video.lingubot.cn`（测试环境）。

```json
  "type": "2010004",
  "data": {
    "status": 0,

  }
}
```

| 字段 | 说明 |
| ---- | ---- |
| data.status | `0` = 关闭推流 |

**响应示例**

```json

  "data": [

      "name": "KRIPC_93002871_100",
```

## 5 远程控制

云平台将第三方下发的控制指令转发至车端 MQTT `dev/sub/{clientId}`。

### 5.1 下发控制指令

| 项 | 值 |
| -- | -- |
| 路径 | `POST /device/instructions` |
| Body | 见下表 |

云平台受理后转发至车端 MQTT `dev/sub/{clientId}`。导航场景（2010001）已验证；监控控制（2010005 / 2010002 / 2010008）**待 M1 联调实测**。

**典型响应**

```json
{
  "code": 0,
  "data": null,
  "msg": ""
}
```

| 说明 |
| ---- |
| `code === 0` 表示云平台已受理；**不表示**车端已执行完毕 |
| 2010002 / 2010008 的车端 `dev/reply` 是否经云同步返回 **待确认** |

**请求 Body**

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| deviceId | String | 是 | 车辆 ID（通常与 MQTT `clientId` 一致；映射规则 **待 Hasun 确认**） |
| type | String | 是 | 指令类型，见下表 |
| data | Object | 是 | 业务数据体 |

**instructions 已支持的 type（HTTP 实测 / 文档）**

| type | 能力 | sztu 状态 |
| ---- | ---- | --------- |
| 2010004 | 开/关摄像头 | ✅ view 1–4 已抓包 |
| 2010002 | 设备控制（灯光/云台/作业等） | ✅ HTTP 受理（如 ultrasonic） |
| 2010005 | 遥控 | ✅ HTTP 受理；车端执行待 M1 联调 |
| 2010008 | 急停 | ⏳ reply 格式待确认 |
| 2010001 | 任务下发 | ⏳ 导航侧称已验证，**本文未展开 Body** |

### 5.2 type = 2010005 遥控车辆

**模式 A（command）**

```json
{
  "deviceId": "LU2605000922",
  "type": "2010005",
  "data": {
    "command": "FORWARD",
    "speedLevel": 2,
    "seq": 1001
  }
}
```

| command | 说明 |
| ------- | ---- |
| FORWARD / BACKWARD / LEFT / RIGHT / STOP | 前进 / 后退 / 左转 / 右转 / 停止 |

**模式 B（speed/angle）** — sztu 环境 HTTP 受理已验证（车端执行待 M1 联调）

```json
{
  "deviceId": "LU2606000100",
  "type": "2010005",
  "data": {
    "speed": 0.1,
    "angle": 0
  }
}
```

**模式 B 通用示例**

```json
{
  "deviceId": "LU2605000922",
  "type": "2010005",
  "data": {
    "speed": 0.6,
    "angle": 0
  }
}
```

| 字段 | 单位 | 说明 |
| ---- | ---- | ---- |
| speed | m/s | 正前进，负后退，限幅 ±1.0 |
| angle | rad | 正左转，负右转，限幅 ±0.524 |

下发频率建议 **10–30 Hz**；松开时发 `STOP`。

### 5.3 type = 2010002 设备控制

**灯光示例**

```json
{
  "deviceId": "LU2605000922",
  "type": "2010002",
  "data": {
    "headlights": 1,
    "turnSignals": 3
  }
}
```

**云台示例**

```json
{
  "deviceId": "LU2605000922",
  "type": "2010002",
  "data": {
    "target": "GIMBAL",
    "command": "MOVE_STEP",
    "params": { "pan": 5.0 }
  }
}
```

**作业示例**

```json
{
  "deviceId": "LU2605000922",
  "type": "2010002",
  "data": {
    "target": "WORK",
    "command": "SET",
    "params": { "action": "SWEEP", "on": true }
  }
}
```

### 5.4 type = 2010008 紧急开关

```json
{
  "deviceId": "LU2605000922",
  "type": "2010008",
  "data": {
    "action": "EMERGENCY_STOP"
  }
}
```

| action | 说明 |
| ------ | ---- |
| EMERGENCY_STOP | 紧急停止 |
| RELEASE | 解除急停 |

### 5.5 指令回复

| type | 云平台同步响应 | 说明 |
| ---- | -------------- | ---- |
| 2010005 | 受理即可 | 车端不要求 MQTT 回复 |
| 2010002 / 2010008 | 返回车端 dev/reply 结果（**格式待 Hasun 确认**） | 含 code、msg |

### 5.6 控制相关 — 待 Hasun 对齐

| # | 项 | 状态 | Hasun 需答复 |
| --- | -- | ---- | ------------ |
| 1 | 2010002/2010008 HTTP 响应 | ⏳ | 仅 `code:0` 受理，还是同步返回车端 `dev/reply` JSON？ |
| 2 | 指令超时 | ⏳ | 车端离线/无 reply 时 HTTP 超时秒数与错误码 |
| 3 | deviceId ↔ clientId | ⏳ | 对照表或规则 |
| 4 | 2010005 车端执行 | ⏳ | M1 联调结论回填 |
| 5 | 2010001 任务下发 | ⏳ | 是否经同一 `instructions` 路径；请求/响应示例 |

## 6 典型调用流程

```
1. POST /system/auth/login → accessToken
2. GET /device/select_device_detail_by_id → 状态 + position（室外高德 / 室内见雷达专项文档）
3. POST /device/instructions type=2010004，view=1..5 分别 status=1 → WebRTC
4. POST /device/instructions type=2010005 / 2010008 / 2010002 → 遥控 / 急停 / 设备控制
5. POST /device/instructions type=2010004，cameraName 逐路 status=0 → 关流
```
