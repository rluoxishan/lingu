> 源文件：[【中性】事件检测算法推理平台-接口定义.pdf](./【中性】事件检测算法推理平台-接口定义.pdf)

> 分析日期：2026-07-01

---

## 1. 这份 PDF 是什么

**鸿羽「事件检测算法推理平台」** 的 HTTP/REST 接口定义，面向 **视频 AI 分析**，不是车端 MQTT 协议。

| 接口类型 | 方式 | 用途 |
| -------- | ---- | ---- |
| 事件任务控制 | HTTP POST/GET/DELETE | 对录像/视频流创建 AI 分析任务 |
| 事件结果推送 | Webhook 回调 | 平台主动 POST 告警到第三方 URL |
| 事件结果查询 | HTTP CRUD | 查询历史告警事件 |
| 相机截图 | HTTP POST | 按 cameraUrl 抓图 |

支持事件包括：火焰、烟雾、跌倒、未戴安全帽、区域入侵、摄像机遮挡等（eventId 14/15/53/121/99…）。

## 2. 与「远程监控与控制」的关系

```
┌─────────────┐   MQTT 2010004    ┌──────────────┐   拉流/GB28181   ┌──────────────────┐
│   车端摄像头  │ ───────────────► │ 流媒体/云平台  │ ───────────────► │ 监控页（实时画面）  │
└─────────────┘                   └──────────────┘                   └──────────────────┘
                                         │
                                         │ 视频 URL / 录像片段
                                         ▼
                                  ┌──────────────────┐
                                  │ 事件检测推理平台    │  ← 本 PDF
                                  │ (AI 分析)         │
                                  └────────┬─────────┘
                                           │ Webhook 推送告警
                                           ▼
                                  ┌──────────────────┐
                                  │ 监控页告警区 /     │
                                  │ 云平台告警服务     │
                                  └──────────────────┘
```

| 维度 | 吉狮 MQTT 协议（我们 Draft） | 本 PDF |
| ---- | --------------------------- | ------ |
| 层级 | **车端 ↔ 控制方**（MQTT） | **视频 ↔ AI 平台**（HTTP） |
| 核心能力 | 推流开关、远程驾驶、灯光云台 | AI 事件识别、告警推送 |
| 是否替代 2010004/2010005 | **否** | **否** |
| 当前阶段优先级 | **P0** | **P2（可选增强）** |

**结论：有参考价值，但不是当前协议定稿和联调的主线文档。**

## 3. 对后续任务的具体参考点

### 3.1 可能有用的部分

| PDF 内容 | 可参考场景 |
| -------- | ---------- |
| `POST /city/camera/snapByUrl` + `cameraUrl`（GB28181 WebSocket） | Hasun 讨论推流/拉流 URL 格式时可对照 |
| `channelId` 国标 ID | 若监控页摄像头与国标平台打通 |
| 事件推送 `CityEventObject`（含 ImgUrl、BigUrl、bbox） | **阶段 4**：监控页「AI 告警」卡片设计 |
| `POST /city/camera/event/video` | 对历史录像做离线 AI 分析（非实时遥控） |

### 3.2 与吉狮 Draft 的差异（勿混用）

| 项 | 吉狮 Draft | 本 PDF |
| -- | ---------- | ------ |
| 控制车端推流 | MQTT `2010004` | 不涉及 |
| 远程驾驶 | MQTT `2010005` | 不涉及 |
| 实时看视频 | 拉 RTMP/WebRTC 等 | 截图 / 分析结果图 URL |
| 认证 | MQTT 密钥 | HTTP JSON，无 MQTT |

### 3.3 摄像机遮挡 eventId=99

若监控页要做 **「视频信号丢失 / 镜头遮挡」** 提示，可复用该平台的遮挡检测，或仅在 UI 层做无信号判断——需产品确认是否要上 AI。

## 6. 关键 API 速查（摘自 PDF）

| 接口 | Method | 路径 |
| ---- | ------ | ---- |
| 添加录像分析任务 | POST | `/city/camera/event/video` |
| 任务进度 | GET | `/city/camera/event/progress?serialnumbers=` |
| 删除任务 | DELETE | `/city/camera/event/video` |
| 相机截图 | POST | `/city/camera/snapByUrl` |
| 事件查询 | GET | `/VIID/CityEvent` |

请求头：`Accept: application/json`、`Content-Type: application/json`
